## unlocker.py

Status: UNSAFE

Summary: Binary patching with obfuscated, potentially malicious payloads

Details:

The script modifies VMware binaries by patching internal structures in `vmware-vmx` and related files, which is a legitimate function for a VMware unlocker (e.g., enabling macOS support on non-Apple hardware). However, it contains a highly suspicious code block that decodes and writes obfuscated strings into the binary using ROT13 encoding:

```python
smc_new_data = codecs.encode('bheuneqjbexolgurfrjbeqfthneqrqcy', 'rot_13')
f.write(smc_new_data.encode('UTF-8'))
```

This decodes to: "osxinstallersguardthepasswoqpl", and a similar line decodes to a string ending with "(c)Apple Computer Inc" — implying forged Apple SMC keys. While this may be part of the intended functionality to bypass macOS restrictions, the broader concern is that the script directly manipulates binary files and injects hidden, obfuscated data payloads into system-critical VMware processes.

Moreover, the script alters relocation entries in ELF/shared objects via `patchelf()` without verifying integrity, and performs undocumented binary patching across multiple platforms (Linux/Windows). These actions, combined with the use of obfuscation, deviate from standard packaging norms and pose a supply chain risk: an attacker could repurpose this script to inject actual malicious payloads instead of just "unlocking" features.

Although some behaviors serve the apparent purpose of the unlocker, the use of obfuscation, low-level binary manipulation, and undocumented patching techniques meets the threshold for an **UNSAFE** rating due to the high potential for abuse and difficulty in auditing runtime impact.
