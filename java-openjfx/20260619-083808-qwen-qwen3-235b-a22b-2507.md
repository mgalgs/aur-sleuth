## crosslibs-armv6hf.sh

Status: UNSAFE

Summary: Downloads unverified compiler and packages over HTTP

Details:

The script downloads and executes a compiler tarball from an HTTP URL hosted on launchpad.net without integrity verification. Specifically, the `installCrossCompiler` function uses:

```
COMPILER_URL=https://launchpad.net/.../gcc-linaro-arm-linux-gnueabihf-raspbian-2012.09-20120921_linux.tar.bz2
CMD="wget $COMPILER_URL -O - | tar jx -C $CROSSLIBS"
```

This pipeline downloads and immediately extracts unverified content directly into the build environment. While the URL appears legitimate, the lack of checksum validation opens a supply-chain risk: a man-in-the-middle attacker or compromised mirror could inject malicious code into the cross-compiler toolchain, which would then be used to build JavaFX for ARM. This undermines the integrity of the entire build process.

Additionally, the script fetches Debian packages over plain HTTP (`http://archive.debian.org/debian/`) and processes them without verification, increasing exposure to tampering. Although many operations are consistent with legitimate cross-compilation setup (e.g. fetching dev libraries), the absence of any signature or hash checking for downloaded binaries crosses from poor hygiene into active risk, especially in a security-sensitive context like building a widely-used runtime component. These issues collectively constitute a genuine supply-chain vulnerability.
