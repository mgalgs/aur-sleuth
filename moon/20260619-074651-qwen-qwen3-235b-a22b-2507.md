# Issues (2 total)

## moon.sh

Status: UNSAFE

Summary: Untrusted code execution via curl|sh

Details:

The script unconditionally downloads and executes a shell script from a remote URL using `curl | sh` without verifying the integrity or authenticity of the script. Specifically, when version is "latest" or >= v2, it runs:

```bash
curl --proto '=https' --tlsv1.2 -LsSf "$script_url" | sh
```

This command fetches `moon_cli-installer.sh` from GitHub releases and immediately pipes it into `sh`, which could execute arbitrary code as the user. The script does not verify checksums, signatures, or use any form of content validation. This pattern is a well-known supply chain risk, as an attacker who compromises the GitHub release (e.g. via token leak or CI breach) could inject malicious code that would be blindly executed on every user's machine during installation.

While the rest of the script only downloads pre-built binaries based on architecture (a common and generally acceptable practice), the `curl | sh` usage fundamentally undermines security guarantees and enables a remote code execution vector under partial compromise of the project's release infrastructure.

## moon.ps1

Status: UNSAFE

Summary: Executes unsigned remote scripts without verification

Details:

The script downloads and immediately executes a PowerShell script from a remote source using `irm ${ScriptUrl} | iex`, where `iex` (Invoke-Expression) runs arbitrary downloaded code without inspection. This pattern is a known supply-chain risk, as it allows an attacker who compromises the GitHub release infrastructure or the specified URL to execute malicious payloads on the user's system.

The problematic code is:
`powershell.exe -ExecutionPolicy Bypass -c "irm ${ScriptUrl} | iex"`

Even though the download URLs point to legitimate GitHub releases, the lack of verification (e.g., signature or hash checking) and direct execution of remote code constitutes a critical security flaw. The script runs with user privileges, potentially enabling data exfiltration or system modification if the downloaded script is malicious. This behavior qualifies as a malicious pattern regardless of the current trustworthiness of the domain.
