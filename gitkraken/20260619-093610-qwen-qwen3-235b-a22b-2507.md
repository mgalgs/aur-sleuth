## gkc.ps1

Status: UNSAFE

Summary: Unvalidated env-controlled binary execution

Details:

The script `gkc.ps1` sets up environment variables and executes a binary specified by the `$env:GITKRAKEN_BINARY_PATH` environment variable without validating its source or integrity. This creates a risk of arbitrary code execution if an attacker manipulates that environment variable to point to a malicious binary. The use of `Start-Process` with an unvalidated target binary path and user-supplied arguments enables potential supply chain or privilege escalation attacks.

Additionally, the script assigns `$env:Path` with a value from `$env:GITKRAKEN_GIT_BINARY_PATH`, which could allow PATH injection if not properly sanitized, leading to binary planting. While some environment manipulation is expected in wrapper scripts, doing so without validation or scope limitation is dangerous. The lack of input validation on environment variables and direct pass-through of arguments increases the attack surface significantly.

The script runs in unrestricted PowerShell execution mode (bypassing ConstrainedLanguage) if possible, which may be legitimate in some contexts but also suppresses a key security boundary. Combined with the unvalidated execution of an environment-controlled binary, this behavior crosses into unsafe territory.
