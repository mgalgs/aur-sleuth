## setup.py

Status: UNSAFE

Summary: Arbitrary code execution via os.system("./Messages.sh")

Details:

The `setup.py` file defines a custom `usb_creator_build_i18n` class that overrides the `run` method to execute `os.system("./Messages.sh")` after the normal i18n build process. This invocation of `os.system` runs an external shell script (`Messages.sh`) without any validation of its existence, contents, or origin. If an attacker can place a malicious `Messages.sh` script in the build directory (e.g., via a compromised or spoofed source tarball, or a supply-chain attack on the source repository), this would result in arbitrary command execution during the build process with the privileges of the user building the package.

While `Messages.sh` may be a legitimate script in the upstream source meant to handle translation extraction for the KDE frontend, the use of `os.system("./Messages.sh")` is inherently risky because it assumes the script is trustworthy without verification. This pattern is commonly exploited in supply-chain attacks where otherwise legitimate build processes are subverted by injecting malicious scripts or binaries into the build context. The lack of hardcoded path, input validation, or integrity checks makes this a vector for unintended code execution.

This issue goes beyond typical packaging risks and constitutes a genuine security flaw because it runs an executable script from the current working directory during build with no safeguards, enabling potential remote code execution if the source tree is compromised.
