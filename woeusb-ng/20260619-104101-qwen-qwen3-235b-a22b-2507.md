## core.py

Status: UNSAFE

Summary: Downloads unverified binary from third-party server

Details:

The file `core.py` contains a security issue in the `install_uefi_ntfs_support_partition` function, where it downloads and executes an external binary image directly from GitHub without integrity verification:

```python
fileName = urllib.request.urlretrieve("https://github.com/pbatard/rufus/raw/master/res/uefi/uefi-ntfs.img", "uefi-ntfs.img")[0]
...
shutil.copy2(download_directory + "/uefi-ntfs.img", uefi_ntfs_partition)
```

This code downloads `uefi-ntfs.img` from a third-party repository (rufus) and writes it directly to a partition on the target device. The downloaded content is not verified using checksums or cryptographic signatures, creating a supply-chain risk. An attacker who compromises the rufus repository or performs a MITM attack could inject malicious code into the image, leading to arbitrary code execution in UEFI firmware. While the download URL is legitimate and related to the tool's purpose, the lack of any validation violates secure software distribution principles.

Although this behavior is documented with a FIXME comment indicating intent to improve, the current implementation remains dangerous as it automatically fetches and installs unsigned binaries from the internet during normal operation.
