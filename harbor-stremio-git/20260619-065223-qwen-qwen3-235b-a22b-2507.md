Although it attempts a fallback to GitHub Releases, the lack of checksum validation or signature checking means there is no protection against supply-chain compromise.

Additionally, the script executes `lib.exe` — a Microsoft tool typically from Visual Studio — based on presence in `PATH`, and runs it with arguments derived from a downloaded `.def` file. If an attacker tampers with `libmpv.def`, this could lead to arbitrary code execution during the `lib.exe` invocation, depending on how `lib.exe` parses its input. The script also automatically copies `libmpv-2.dll` into build output directories, potentially enabling malicious DLL sideloading if the binary is compromised.

These behaviors — downloading and extracting unsigned, unverified binaries from a CI pipeline, using them in build/link processes, and loading them into the output binary — represent a significant supply-chain risk. The script essentially trusts the `shinchiro/mpv-winbuild-cmake` repository's CI pipeline completely, which goes beyond normal dependency handling and into potentially dangerous territory.
