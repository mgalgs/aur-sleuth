## install_biometrics_policy.sh

Status: UNSAFE

Summary: Root privilege escalation via unsafe eval usage

Details:

The script uses `eval` with a heredoc to process a template file (`com.1password.1Password.policy.tpl`), interpolating the `POLICY_OWNERS` variable via command substitution and `eval`. This pattern is dangerous because `eval` can lead to arbitrary code execution if the template or input data is tampered with. Specifically, the line `eval "cat <<EOF\n$(cat ./com.1password.1Password.policy.tpl)\nEOF"` will execute any shell commands present in the template file as code, not just treat them as text. For example, if the template contains `$(rm -rf /)` or similar, it would be executed with root privileges.

Additionally, the script runs with root privileges (as required by PolKit policy installation) and does not validate or sanitize the local template file before processing it. This creates a supply-chain risk: an attacker who can modify the `.tpl` file — for example, via a compromised download or malicious contributor — could inject arbitrary commands that execute as root during installation. This constitutes a clear vector for privilege escalation and system compromise.

Although PolKit policy installation is a legitimate need for some applications, the use of `eval` on untrusted or externally supplied templates is a well-known anti-pattern and security hazard. Safer alternatives like `sed` or dedicated template engines should be used instead.
