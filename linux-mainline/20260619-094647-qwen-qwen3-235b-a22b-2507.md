## autoload.sh

Status: UNSAFE

Summary: Modifies system config insecurely, risks privilege escalation

Details:

The script writes to the system configuration file `/etc/modprobe.d/aoe.conf` without proper privilege checks or safeguards. It appends module alias directives (`alias block-major-152 aoe` and `alias char-major-152 aoe`) directly into the file if they are not already present, but it does so using unquoted variable expansion and unvalidated file access.

The conditions `test ! -r $f || test ! -w $f` check readability and writability but do not prevent race conditions — an attacker could replace `$f` between the check and the write. Additionally, the script runs commands that require root privileges (writing to `/etc/modprobe.d/`) without any mechanism to verify or prompt for elevation, meaning it is likely intended to be run as root. This increases the risk of privilege escalation if the script is manipulated or the file is symlinked.

Most critically, it modifies system module loading behavior, which can lead to unauthorized kernel module loading or persistence mechanisms. While the intent appears to be legitimate (enabling AoE module autoloading), the insecure file handling and lack of atomic, safe file updates make this script a vector for system compromise.
