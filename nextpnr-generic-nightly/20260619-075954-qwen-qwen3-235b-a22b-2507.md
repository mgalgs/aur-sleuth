## picorv32.sh

Status: UNSAFE

Summary: Downloads and processes unverified remote hardware code

Details:

The script downloads a Verilog file (`picorv32.v`) from an external source using `wget` without any checksum verification or integrity check. The URL points to a raw GitHub content path (`https://raw.githubusercontent.com/cliffordwolf/picorv32/master/picorv32.v`), which refers to the mutable `master` branch. This means the fetched content can change at any time without detection, enabling potential supply-chain attacks if the upstream repository is compromised.

More critically, the script directly feeds the downloaded `picorv32.v` into `yosys` and other tooling, effectively executing hardware synthesis on unverified, remotely fetched code. While the tools themselves are legitimate, the lack of any validation or pinning to a specific commit or version allows for arbitrary code (or malicious hardware logic) to be processed, which could lead to unexpected or malicious behavior in the resulting FPGA bitstream. The use of `set -ex` ensures that any failure will be visible, but it does not mitigate the risk of malicious payloads.

This script exhibits a clear supply-chain vulnerability: it automatically downloads and processes executable-like design artifacts from an unpinned, unverified remote source. This is unsafe in a packaging or build environment where reproducibility and integrity are required.
