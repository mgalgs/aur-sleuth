## build-linux.sh

Status: UNSAFE

Summary: Dangerous unverified Docker image usage with host volume mount

Details:

The script `build-linux.sh` executes two `docker run` commands that pull and run containers from the `fritzing/build:bionic` image. It mounts the current working directory (`$(pwd)`) as a volume into the container at `/home/conan/fritzing` and runs `qmake` and `make` inside that environment.

The critical issue is that the Docker image `fritzing/build:bionic` is not built locally or pinned to a known digest; it is pulled implicitly from a remote registry (likely Docker Hub) by tag. An attacker who has compromised the `fritzing/build` Docker image could inject arbitrary code into the build environment. Since the build process runs with the host's mounted filesystem, malicious actions inside the container—such as modifying source files, exfiltrating data, or executing harmful scripts—could impact the host system or compromise the build artifact.

While `qmake` and `make` are legitimate build tools, the lack of image verification and the use of a mutable tag (`bionic`) from an external source introduce a supply-chain risk. Combined with the volume mount that allows full read/write access to the project directory, this creates a potential vector for code injection or data exfiltration if the image is compromised.
