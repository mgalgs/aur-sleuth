## install-rpm.sh

Status: UNSAFE

Summary: Unsafe remote execution with root privileges and untrusted sources

Details:

The script directly downloads and executes code from a third-party GitHub Pages URL (`https://patrickjaja.github.io/`) without verification or transparency. It uses `curl -fsSL ... | sudo bash` in its usage instructions, which is a well-known anti-pattern that enables remote code execution as root with no auditability. The script itself lacks integrity checks or versioning, and the use of `sudo bash` to run unversioned, unsigned scripts from a personal GitHub Pages site creates a critical supply-chain risk. An attacker who compromises the `patrickjaja` GitHub account could silently replace the script and gain root execution on all users' systems.

Additionally, the script imports a GPG key from the same untrusted origin (`$REPO_URL/gpg-key.asc`) and uses it to sign packages from the same domain. There is no out-of-band mechanism to verify the authenticity of this key. This constitutes a single point of failure: if the domain or GitHub Pages site is compromised, an attacker can sign and distribute malicious RPMs that will be trusted and installed system-wide. The combination of root privileges, unsigned/unsigned sources, and direct execution from an arbitrary web host constitutes a high-risk attack vector.

This script's behavior — downloading and executing unsigned code as root from a mutable personal website — is consistent with supply-chain attack patterns. While the current code *appears* to only set up a repository, the trust model is fundamentally broken. The usage instruction itself (`curl | sudo bash`) is inherently unsafe and should never be used in production environments.
