## battery_status_helper.sh

Status: UNSAFE

Summary: Enables unauthenticated system-level CPU governor manipulation

Details:

The script writes command-line input directly to kernel interface files controlling CPU frequency scaling, allowing unprivileged users to modify system behavior globally. When executed with an argument like 'performance' or 'powersave', it broadcasts that setting to all CPU cores by writing to /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor. While CPU governor control is legitimate for system tools, this script provides no authentication, validation, or access control and is world-writable or executable depending on packaging. An attacker could use it to force high-performance modes to degrade battery life or induce thermal stress, or conversely disable performance when needed. More critically, because it lacks any check on the input value, it allows injection of arbitrary strings into the kernel interface, which may lead to denial of service or potentially exploitation if malformed values are accepted. Such functionality should require root privileges and explicit authorization, but the script runs without safeguards, making it a potential vector for privilege escalation or system manipulation if accessible from untrusted contexts.
