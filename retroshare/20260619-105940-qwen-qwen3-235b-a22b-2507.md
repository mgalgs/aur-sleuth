## make.sh

Status: UNSAFE

Summary: Uses Debian tooling on Arch package, unsafe.

Details:

The script makes unauthorized and potentially malicious use of hardcoded paths and commands that suggest a supply-chain attack. Specifically, it copies built `.deb` packages from `/home/csoler/pbuilder/...` to the current directory, which is not standard packaging practice and indicates the script is tied to a specific, non-systematic build environment. More critically, it uses `pbuilder-dist` commands in a loop across multiple Ubuntu distributions (e.g., karmic, lucid) — this is highly unusual for an Arch Linux AUR package, as pbuilder is a Debian/Ubuntu toolchain, not used in Arch packaging.

The fundamental red flag is that this script is not a PKGBUILD or Arch-compliant build script, yet it is included in a context that may mislead users into running it. It clones behavior typical of a malicious wrapper script: removing directories, rebuilding source packages via SVN (with no checksum or verification), and using hard-coded home paths (`/home/csoler`). These are hallmarks of a backdoored or hijacked build process. The use of SVN without integrity checks and the silent copying of artifacts to the current directory could enable binary exfiltration or trojan injection.

```sh
cp /home/csoler/pbuilder/"$dist"_result/retroshare_0.5.3-0."$svn"~"$dist"_amd64.deb .
```
This line copies pre-built Debian packages from a personal directory, which could be controlled by an attacker, risking the distribution of malicious binaries. The fact that the script operates outside standard AUR tooling (makepkg, PKGBUILD) and leaks Debian-centric logic strongly indicates it is unsafe and potentially repurposed for malicious builds.

## full_packaging_pipeline.sh

Status: UNSAFE

Summary: Wildcard DSC filename allows malicious package injection.

Details:

The script `full_packaging_pipeline.sh` contains a suspicious command that downloads and executes code from an unspecified source using `pbuilder-dist`, a tool used to build Debian packages in clean chroots. While `pbuilder-dist` itself is a standard packaging tool, the script does not define or constrain its configuration, potentially allowing execution of untrusted or modified build environments.

More critically, the script constructs the DSC filename using a glob pattern: `retroshare.6.2-1.???????.????????~"$dist".dsc`. The use of a wildcard (`???????`) in the version portion of the filename is highly unusual and dangerous. It implies that the script will match *any* DSC file matching the pattern in the current directory, which could be manipulated by an attacker to substitute a malicious source package. This introduces a supply-chain risk where an attacker could place a crafted `.dsc` file to trigger unintended builds or potentially execute code during the build process.

Additionally, the script calls `./makeSourcePackage.sh` with command-line arguments including unescaped variables, which may lead to injection risks if those variables are derived from untrusted input. The lack of explicit version pinning or integrity checks on generated source packages reduces auditability and enables version confusion attacks.
