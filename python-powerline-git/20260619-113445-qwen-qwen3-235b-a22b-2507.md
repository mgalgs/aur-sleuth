This weakens supply-chain security: the commit 6f8da88 could contain malicious changes, and there is no check on its origin or integrity. - Is not part of the declared upstream branch (`source`)
- Has no accompanying checksum or verification
- Could silently introduce malicious behavior (e.g., backdoors, telemetry, data exfiltration)
Even if the commit is benign today, this pattern enables a supply-chain compromise if the maintainer's Git history is spoofed or if the commit is later swapped in a forked repo.

Thus, despite otherwise standard packaging structure, this unverified code injection makes the package unsafe.
