## bash-0.3.3.hook

Status: UNSAFE

Summary: Exfiltration risk via environment snapshot in background

Details:

The provided script is a Bash hook for a tool called `tirith`, which appears designed to intercept and control command execution in interactive shells. While much of the functionality relates to security enforcement (e.g., blocking suspicious commands, approval workflows), it exhibits **genuinely malicious characteristics** due to its **capability to exfiltrate potentially sensitive data**.

Specifically, in the section marked "M9 ch4", the script launches a backgrounded subprocess on every interactive shell start:

```bash
if [[ $- == *i* ]]; then
  command tirith env snapshot >/dev/null 2>&1 &
  disown 2>/dev/null || true
fi
```

This executes `tirith env snapshot`, which is described as recording a "shell-start environment snapshot" and writing it to `<state-dir>/env_snapshot.json`. Crucially, while the description claims only "variable names + an 8-char value-hash prefix" are stored (and not "raw values"), this is **self-reported assurance, not verifiable behavior**. The actual behavior of the `tirith` binary is **not visible in this script**, meaning the binary could **easily be modified to exfiltrate the full environment variables**, including secrets like `SSH_AGENT_PID`, `GNUPGHOME`, API keys, tokens, or custom environment variables containing credentials.

Environment variables are a common source of secrets in shell sessions. Running an untrusted or compromised binary that reads the full environment — even if only hashing values — represents a **privilege escalation and data exposure risk**. The fact that this runs in the background, detached from the user's control (`disown`), and executes silently without explicit consent each time a shell starts, makes this behavior particularly dangerous.

Even if the current version of `tirith` does not exfiltrate data, this hook enables such capability. A malicious update to the `tirith` binary could silently begin sending environment snapshots (including full values) to a remote host, turning this into a **persistent credential-harvesting backdoor**.

Therefore, due to the **execution of a privileged external binary that reads the full shell environment and stores a hashed snapshot, combined with background, silent, and persistent execution**, this file is marked **UNSAFE**.
