Status: UNSAFE

Summary: Installs a systemd service without origin validation or safeguards

Details:

The script creates and installs a systemd service file that points to a launcher script (`jackett_launcher.sh`) located in the same directory as the binary. While setting up services for an application is normal, the script makes assumptions about ownership and file locations that could be exploited if an attacker gains partial access.

Most critically, the script checks the owner of `./jackett` to determine the unprivileged user for running the service, but then uses that same user for the service configuration *without validating or sanitizing the path or ownership chain*. It also proceeds to install and start the service without confirming the origin or integrity of the `jackett` binary or its launcher. The `ExecStart` points to `/bin/sh "${JACKETT_DIR}/jackett_launcher.sh"`, which could be a wrapper script. If an attacker can modify or inject this script (e.g., via symlink, file replacement), the service would execute arbitrary code as the configured user.

Although this script may be intended for manual use after building Jackett from source, running it directly from an untrusted or compromised build directory grants persistent execution via systemd. Such installation scripts are common attack vectors in AUR-like ecosystems when they lack checksums, origin validation, or user confirmation steps. This script, when executed as root, grants privilege persistence and could be abused in a supply chain attack if bundled with malicious or tampered binaries — making it unsafe.
