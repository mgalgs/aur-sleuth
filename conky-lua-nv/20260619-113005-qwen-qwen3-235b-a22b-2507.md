## build.sh

Status: UNSAFE

Summary: Downloads and executes unsigned, mutable AppImages without verification

Details:

The script downloads and executes two binaries (`linuxdeploy-x86_64.AppImage` and `appimagetool-x86_64.AppImage`) from GitHub releases using `wget` without verifying their integrity or authenticity. Although the URLs are from GitHub.com, no checksum validation or signature verification (beyond optionally signing the final AppImage) is performed on these tools before execution. This creates a supply-chain risk: an attacker who compromises the GitHub account or repository could serve malicious binaries that escalate privileges, exfiltrate data, or tamper with the build process.

Furthermore, executing downloaded binaries via `./` immediately after `chmod +x` constitutes a high-risk pattern. While GPG signing of the final output is attempted, the build tools themselves are blindly trusted. No `sha256sum` check or pinned version is used — the script fetches the "continuous" (i.e., latest development) builds, which are mutable and unversioned. This undermines reproducibility and increases exposure to malicious code injection.

The script also repeats `-DBUILD_IRC=ON`, which is benign but suggests sloppiness. However, the core issue is the unsafe download-and-execute pattern involving externally fetched binaries. This deviates from secure packaging practices and introduces a genuine risk of remote code execution during the build.
