## download.py

Status: UNSAFE

Summary: Downloads unverified files from Google Drive, high supply-chain risk

Details:

The `gdrive_download` function in this script downloads files from Google Drive without any validation of the file's integrity or authenticity beyond what is provided internally by the script. While the script includes a `sha1sum` function, it is not used within the `gdrive_download` function to verify the downloaded content, leaving the door open for potential supply-chain attacks if the Google Drive file is tampered with or swapped.

More critically, the script is designed to fetch arbitrary files from Google Drive using a file ID and automatically execute no further verification before saving to disk. If this script is used in a PKGBUILD or build process to retrieve binaries or code that is later executed, it represents a significant risk. Google Drive URLs are not version-controlled, can be silently replaced, and are not amenable to standard audit practices. This makes them an unsuitable source for reproducible, secure package builds.

Additionally, the use of `urllib.request` with dynamic URL construction and automatic cookie handling could be abused if the `gdrive_id` is derived from untrusted input. Although no explicit malicious behavior (like exfiltration) is present, the design enables high-risk operations common in supply-chain compromises—especially in AUR packages where users expect scrutinized, verifiable sources.

```python
req = urllib.request.Request(f'{url}&confirm={confirm}')
with urllib.request.urlopen(req, timeout=10) as resp:
    filename = get_filename(resp.getheaders())
    with open(os.path.join(path, filename), 'wb') as save_file:
        save_file.write(resp.read())
```
This snippet downloads and saves a file without hash verification, executed silently and directly to disk.
