## pack-resources.sh

Status: UNSAFE

Summary: Unverified remote code download and packaging

Details:

The script downloads a zip archive from the JetBrains/phpstorm-stubs repository using a dynamically fetched commit hash from the GitHub API and immediately executes operations that extract, move, and process the contents without verifying integrity or authenticity. The use of `curl -L -o $ZIP_FILE $ZIP_URL` followed by `unzip` directly into the project structure allows for remote code execution if the upstream repository is compromised or spoofed.

More critically, the script later deletes all files in `$PHPSTORM_STUBS_DIR` that are not directories (`find $PHPSTORM_STUBS_DIR -maxdepth 1 -type f -delete`), then proceeds to create a new `PHP.zip` archive containing `TestXDebugSettings.php` and the `cc/` directory — and finally removes the stubs directory and version file. These operations suggest transient resource handling, but the entire `cc/` folder is archived and shipped as part of the output, potentially allowing an attacker who compromises the stubs repo to inject code that ends up packaged and distributed.

There is no checksum validation, no signature checking, and the source is a mutable branch (master), making this susceptible to supply chain injection. The combination of unverified download, execution via inclusion in a runtime archive, and lack of any integrity controls constitutes a genuine security risk.
