## libraryoptionseditor.js

Status: UNSAFE

Summary: Hardcoded tracking link to untrusted external domain

Details:

The JavaScript file `libraryoptionseditor.js` contains a suspicious hardcoded URL pointing to a SendGrid tracking domain (`https://u10505776.ct.sendgrid.net/ls/click?upn=...`) within a string that is rendered as HTML content when a specific plugin ("thetvdb") is processed. This URL is embedded in a call to `servicelocator.appHost.supports()` and is used in a link that promotes TheTVDB.

While some metadata attribution links are normal, this specific URL is a third-party tracking link from SendGrid, a service commonly used for email campaigns and analytics—not typically used for legitimate project attribution. This domain is not the official TheTVDB website, nor is it a known partner domain. Instead, it is capable of redirecting users through a tracking endpoint, which raises serious supply-chain concerns.

Given that this link is hardcoded into the AUR package's front-end JavaScript and not pulled dynamically from a trusted API, and considering its use of a third-party tracking domain with potential for link rotation or payload injection, this constitutes a potential data exfiltration or phishing risk. It deviates from standard open-source attribution practices and introduces an untrusted network destination within the application’s UI logic.

This behavior qualifies as malicious under supply-chain security standards because it communicates with an unexpected and untrusted external endpoint from within a packaged component.
