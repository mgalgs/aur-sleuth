## picorv32.sh

Status: UNSAFE

Summary: Fetches and processes unsigned remote HDL code

Details:

The script downloads a Verilog file (`picorv32.v`) directly from `https://raw.githubusercontent.com/cliffordwolf/picorv32/master/picorv32.v` using `wget` without verifying its integrity or authenticity. This represents a supply-chain risk, as the file is not pinned to a specific commit or version and could be silently modified if the upstream repository is compromised.

More critically, the script immediately feeds this externally fetched, unverified hardware description into `yosys` and `nextpnr-ice40` tools for synthesis and place-and-route. While these are legitimate FPGA toolchain components, executing synthesis on unchecked, remotely fetched HDL code could potentially trigger vulnerabilities in the toolchain (e.g., parser exploits in yosys). Though no explicit exfiltration or obfuscated payload is visible, the pattern of fetching and processing unsigned, mutable upstream content without checksum verification constitutes an unacceptable security risk in a packaging context.

The script also executes `../nextpnr-ice40` from a relative path, which could allow execution of an unintended or malicious binary if the directory structure is manipulated. Combined, these behaviors deviate from safe packaging practices and introduce unnecessary trust assumptions.
