## install.sh

Status: UNSAFE

Summary: Downloads unverified code with elevated privileges

Details:

The script performs a direct download and installation of icon themes from GitHub without verifying cryptographic signatures or checksums. It fetches the archive from the `master` branch using `wget` and immediately unpacks and installs it with elevated privileges via `sudo` if needed. This creates a supply-chain risk because the `master` branch is mutable and unverified.

More critically, the script downloads and executes code from an untrusted source (GitHub) without integrity checks (e.g., no signature or hash verification), allowing a potential attacker who compromises the repository or account to deliver a malicious payload. The use of `sudo` in `_sudo` to elevate `cp`, `gtk-update-icon-cache`, and other operations increases the impact of such a compromise.

Although the script appears to be for legitimate use and the actions resemble standard installation steps, the complete lack of validation on downloaded content—combined with automatic privilege escalation—constitutes a dangerous pattern commonly exploited in supply-chain attacks.

_msg "Unpacking archive ..."
tar -xzf "$temp_file" -C "$temp_dir"
