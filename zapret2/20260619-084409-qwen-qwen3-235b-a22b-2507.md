## get_reestr_preresolved_smart.sh

Status: UNSAFE

Summary: Downloads untrusted blocklists with disabled SSL verification

Details:

The script `get_reestr_preresolved_smart.sh` downloads compressed data from `https://raw.githubusercontent.com/bol-van/rulist/main`, a third-party external source unrelated to the package itself. The URLs (`reestr_smart4.txt`, `reestr_smart6.txt`) point to files controlled by the `bol-van/rulist` GitHub repository, which is not an official or verifiable government or public list source. The script uses `curl` with the `-k` flag (disabling SSL certificate verification), weakening transport security and allowing potential MITM attacks. While `--max-filesize` and size checks are present, the origin of the data is untrusted and arbitrarily executable logic could be embedded in downstream processing (e.g. by `create_ipset.sh`). Worse, the `zzcopy` command (not a standard utility) is used to copy the downloaded content, implying the presence of a custom, possibly obfuscated or unreviewed decompression or processing routine elsewhere in the package. This chain enables potential remote code execution or data exfiltration if the fetched lists are interpreted in a dangerous way. The use of unverified, mutable remote content combined with non-standard tooling and disabled certificate validation constitutes a supply-chain risk.

A red flag is the line: `curl -H "Accept-Encoding: gzip" -k --fail --max-time 120 ...` — the `-k` flag is unnecessary and dangerous in a security context, indicating a lack of trustworthiness in the communication layer. The source `bol-van/rulist` is not documented as the canonical source for any official blocklist, and no checksums or signature verification are performed on the downloaded content. These factors together elevate this from a benign updater to a potential vehicle for malicious payloads.
