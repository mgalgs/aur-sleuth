## cargo-vendor-archive.sh

Status: UNSAFE

Summary: Command injection risk via untrusted directory and eval use

Details:

The script contains a critical security flaw in the `cd ${TMP_DIR}/*` line, which changes into a subdirectory whose name is derived from untrusted input (the contents of the extracted archive). This path is then used to execute `eval ${CARGO} vendor --locked`, where `CARGO` is set to `cargo` but could be influenced if the environment is manipulated.

More critically, the use of `eval` on a command involving untrusted input creates a command injection risk. Although `cargo` is a known binary, the `eval` usage is unnecessary and dangerous — especially since the working directory is attacker-controlled via the archive contents. An attacker crafting a malicious archive could potentially influence environment or use symbolic links to manipulate execution context, especially if this script runs with elevated privileges.

Additionally, the script instructs users to upload the resulting archive to a VideoLAN FTP server and manually add a checksum to a source repository — behavior that, while possibly legitimate in a controlled maintainer workflow, could facilitate a supply-chain attack if the script were used in an automated context or if the archive processing were tampered with.
