## build-ui-linux.sh

Status: UNSAFE

Summary: Installs system packages via sudo; not safe for Arch

Details:

The script `build-ui-linux.sh` contains commands that perform unauthorized system modifications outside the scope of a standard AUR build process. Specifically, it uses `sudo apt update` and `sudo apt install -y libayatana-appindicator3-dev` to modify the system package database and install new packages without user interaction. This violates AUR packaging guidelines, which require that all dependencies be declared in the PKGBUILD and handled by the package manager (e.g., pacman or an AUR helper), not directly manipulated at build time.

Additionally, the use of `sudo` within a build script poses a significant security risk, as it grants elevated privileges during an unattended operation. Build scripts should not alter the system's state — they should only compile and package software. The installation of arbitrary packages via `apt` also indicates that this script may not even be intended for Arch Linux (which uses `pacman`, not `apt`), further suggesting it's either misdirected or maliciously crafted to run on a non-target system.

This constitutes a supply chain attack vector: if executed, it could silently modify system packages, potentially introducing backdoors or tampering with system integrity. The presence of such code in a file distributed under the guise of an Arch package is highly suspicious and dangerous.

## ui-post-install.sh

Status: UNSAFE

Summary: Dynamically restarts UI as inferred user via unsafe su pattern

Details:

The script retrieves the PID of a running `netbird-ui` process and obtains the user ID (`uid`) associated with it, either from `/proc/$pid/loginuid` or via `stat`. It then uses `id -nu` to get the username and subsequently executes `su - ${username} -c 'nohup /usr/bin/netbird-ui > /dev/null 2>&1 &'` to restart the process under that user.

This pattern is problematic: it dynamically determines a user context based on a possibly untrusted runtime state and uses `su` to spawn a process as that user without securely validating the user identity or sanitizing the execution environment. More critically, the use of `pgrep -f` with a partial match (`/usr/bin/netbird-ui`) makes it susceptible to matching unintended processes if the name or path can be influenced. Additionally, re-launching a privileged or user-facing binary via `su` in a post-install script—especially without confirming session context—can introduce security escalation vectors or persist processes in unexpected contexts.

While restarting a UI component after an update may be intended, doing so by inferring the user via `/proc` metadata and invoking `su` with dynamic parameters crosses into risky behavior, especially in a post-install script that may run with elevated privileges. The lack of input validation and the use of fragile process identification increase the risk of misuse or unintended privilege delegation.
