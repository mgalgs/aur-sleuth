## reinstall_debs.sh

Status: UNSAFE

Summary: Dangerous unvalidated system-wide reinstall command

Details:

The script `reinstall_debs.sh` contains a dangerous one-liner that rebuilds a package list from `dpkg -S LC_MESSAGES` and uses it as input to `apt-get --reinstall`. While the intent may be to reinstall packages providing locale data for the `LC_MESSAGES` locale, the approach is fragile and potentially exploitable.

The command `dpkg -S LC_MESSAGES` searches for installed packages that own files with paths containing the string "LC_MESSAGES". This string match is not limited to actual locale files and could incorrectly match unrelated packages with "LC_MESSAGES" in file paths or script content. The output is then parsed using `cut`, `tr`, and `sort` to extract package names, which are passed directly to `apt-get --reinstall`. If any malicious or unintended package is falsely detected as owning a file path with "LC_MESSAGES", it would be reinstalled without user confirmation.

Additionally, running `apt-get` with `-u`, `--reinstall`, and `--fix-missing` in an unattended way escalates privileges and performs system-wide modifications. While `apt-get` requires root privileges, the script itself does not validate the input or sanitize the package list, making it unsafe for automated use. The lack of input validation and reliance on string matching for critical system changes constitutes a security risk.

## localepurge.diff

Status: UNSAFE

Summary: Introduces untrusted configuration binary path

Details:

The provided diff modifies the `localepurge` script in a way that introduces suspicious behavior. While the majority of changes are benign (formatting, updated messages, and logic simplification), the critical red flag lies in the replacement of `dpkg-reconfigure localepurge` with `/usr/bin/localepurge-config` in the help and informational output. This implies that a new binary—`/usr/bin/localepurge-config`—is expected to exist and be used for configuration. However, **no such binary is referenced in any standard `localepurge` distribution**, and the diff provides no source or justification for it.

This is highly suspicious because it silently replaces a well-known, auditable system command (`dpkg-reconfigure`) with an opaque custom script that could perform arbitrary actions. The absence of this binary in the `diff` or accompanying package files suggests it may be introduced through another vector, such as a separate download or post-install script—not shown here but potentially part of the full package. This pattern is consistent with supply chain attacks where trust is subverted by redirecting administrative actions to malicious binaries.

Although the diff itself does not contain overtly malicious code (like `curl|bash` or data exfiltration), it lays the groundwork for such attacks by establishing a trust path to an undocumented, unreviewed executable. This weakens system integrity and could allow persistent, stealthy control over locale configuration—potentially enabling privilege escalation or backdoor installation. The change is not aligned with standard `localepurge` behavior and deviates from expected AUR packaging practices without justification.

```diff
-    echo "    dpkg-reconfigure localepurge"
+    echo "    /usr/bin/localepurge-config"
```
